## प्रस्तावना

शुरुआत में, कार्य-प्रमाण (Proof of Work) था। खनिक बिजली जलाकर हैश की गणना करते थे,
और नेटवर्क उन्हें सिक्कों से पुरस्कृत करता था। यह काम सुरक्षा के लिए उपयोगी था,
लेकिन मानवता के लिए बेकार।

टैक्सोइन एक मूलभूत समस्या को हल करने के लिए बनाया गया था:

> **बेकार गणना को उपयोगी योगदान से बदलें।**

"एक CPU, एक वोट" (नाकामोटो, 2008) नहीं।
बल्कि **एक मनुष्य, एक योगदान, एक मूल्य।**

## भाग I: टैक्सोइन क्या है?

### I.1 परिभाषा

**टैक्सोइन (Ⓣ)** टैक्सोइन नेटवर्क में **सिद्ध योगदान** की एक इकाई है।
इसे खनन नहीं किया जा सकता, ढाला नहीं जा सकता, या शून्य से नहीं बनाया जा सकता।

प्रचलन में प्रत्येक Ⓣ नेटवर्क में दो द्वारों में से एक के माध्यम से आया:

| द्वार | तंत्र | अधिकतम आपूर्ति |
|------|-----------|------------|
| **उत्पत्ति (Genesis)** | व्यक्तित्व का प्रमाण (Proof of Personhood) — एक मनुष्य, एक उत्पत्ति | 21,000,000 Ⓣ (420,000 लोग × 50Ⓣ) |
| **सेवा (Service)** | योगदान का प्रमाण (Proof of Contribution) — वास्तविक सेवा प्रदान करना | असीमित (वास्तविक आर्थिक गतिविधि द्वारा सीमित) |

### I.2 मूल सिद्धांत

> **बिना संबंधित योगदान के कोई टैक्सोइन नहीं है।**

यह मौलिक कानून है। प्रत्येक सिक्का इसके द्वारा समर्थित है:
- एक अद्वितीय मनुष्य (उत्पत्ति), या
- एक सत्यापित सेवा प्रदान की गई (योगदान)

कोई खनिक बिजली से सिक्के नहीं निकाल रहा है। कोई स्टेकर निष्क्रिय आय
नहीं कमा रहा है। केवल **वह काम है जिसे किसी ने भुगतान करने के लिए
पर्याप्त मूल्यवान समझा।**

## भाग II: उत्पत्ति (Genesis) — व्यक्तित्व का प्रमाण (Proof of Personhood)

### II.1 बूटस्ट्रैप समस्या

सेवा अर्थव्यवस्था शून्य से शुरू नहीं हो सकती:
- किसी के पास सेवाओं के लिए भुगतान करने के लिए सिक्के नहीं हैं
- कोई भी सेवाएं प्रदान किए बिना सिक्के नहीं कमा सकता
- गतिरोध

### II.2 समाधान: एक मनुष्य, एक उत्पत्ति

टैक्सोइन उत्पत्ति सिक्के "खनिकों" या "निवेशकों" को नहीं,
बल्कि सत्यापनकर्ता समूह द्वारा सत्यापित **अद्वितीय मनुष्यों** को वितरित करता है।

```
वैश्विक पैरामीटर:
  GENESIS_REWARD     = 50 Ⓣ  (प्रति अद्वितीय व्यक्ति)
  MAX_GENESIS_SUPPLY = 21,000,000 Ⓣ
  MAX_PARTICIPANTS   = 420,000  (21M / 50)
```

### II.3 उत्पत्ति समारोह

```
  ┌──────────────────────────────────────────────────────────────┐
  │                उत्पत्ति समारोह (GENESIS CEREMONY)            │
  │                                                              │
  │  1. मनुष्य एक कुंजी जोड़ी उत्पन्न करता है (secp256k1)        │
  │     → पता: 0xdeadbeef...                                     │
  │                                                              │
  │  2. मनुष्य एक सत्यापनकर्ता से संपर्क करता है (व्यक्तिगत रूप    │
  │     से / वीडियो कॉल / विश्वसनीय रेफरर के माध्यम से)          │
  │     → साबित करता है कि वे एक अद्वितीय मनुष्य हैं             │
  │     → अपनी कुंजी से एक संदेश पर हस्ताक्षर करता है:           │
  │       "मैं वास्तविक हूं"                                      │
  │                                                              │
  │  3. सत्यापनकर्ता जांचता है:                                  │
  │     → हस्ताक्षर दावा किए गए पते से मेल खाता है              │
  │     → मनुष्य पहले से उत्पत्ति रजिस्ट्री में नहीं है         │
  │     → मनुष्य एक वास्तविक अद्वितीय व्यक्ति प्रतीत होता है    │
  │                                                              │
  │  4. सत्यापनकर्ता एक प्रमाणन पर हस्ताक्षर करता है:           │
  │     "0xval1 प्रमाणित करता है कि 0xdeadbeef एक अद्वितीय      │
  │      मनुष्य है"                                               │
  │                                                              │
  │  5. 3-में-से-7 सत्यापनकर्ता प्रमाणन आवश्यक → उत्पत्ति स्वीकृत│
  │     → +50 Ⓣ 0xdeadbeef को जमा                              │
  │     → पता उत्पत्ति रजिस्ट्री में जोड़ा गया (अपरिवर्तनीय)    │
  │                                                              │
  └──────────────────────────────────────────────────────────────┘
```

## भाग III: सेवा अर्थव्यवस्था — योगदान का प्रमाण (Proof of Contribution)

### III.1 सेवा क्या है?

सेवा कोई भी योगदान है जो एक नेटवर्क भागीदार दूसरे को प्रदान करता है।
सेवा **श्रृंखला पर पंजीकृत** होती है और **प्रदाता द्वारा मूल्य निर्धारित**
की जाती है।

```
  सेवा प्रकार            प्रदाता              मूल्य
  ─────────────────────────────────────────────────
  SMS गेटवे             एलिस का फोन        0.1 Ⓣ/SMS
  GPU कंप्यूट          बॉब का फार्म       5.0 Ⓣ/घंटा
  API पहुंच            कैरल का मॉडल      0.5 Ⓣ/कॉल
  सवारी (टैक्सी)        डेव की कार         3.0 Ⓣ/किमी
  फ़ाइल भंडारण         ईव का सर्वर        0.01 Ⓣ/MB/माह
  CPU चक्र             फ्रैंक का डेस्कटॉप 0.001 Ⓣ/CPU-घंटा
  बैंडविड्थ रिले       ग्रेस का राउटर    0.05 Ⓣ/GB
  मानव समीक्षा         हैंक की विशेषज्ञता 10.0 Ⓣ/समीक्षा
```

### III.2 सेवा रजिस्ट्री (Service Registry)

प्रत्येक सेवा श्रृंखला पर git कमिट के रूप में पंजीकृत होती है:

```python
@dataclass
class ServiceRegistration:
    provider: str              # पता (0x...)
    service_type: str          # "sms", "gpu", "taxi", ...
    price_per_unit: float      # Ⓣ प्रति इकाई
    description: str           # मानव-पठनीय
    endpoint: str              # कैसे कॉल करें (URL, संपर्क, API)
    attested_by: list[str]     # सत्यापनकर्ता जिन्होंने जांचा
    rating: float = 0.0
    total_tx: int = 0
    created_at: float
```

### III.3 मूल्य निर्धारण

प्रदाता मूल्य निर्धारित करता है। बाजार इसे नियंत्रित करता है।

```
  उच्च मूल्य + खराब सेवा → कोई ग्राहक नहीं → प्रदाता मूल्य घटाता है
  कम मूल्य + अच्छी सेवा → कई ग्राहक → प्रदाता प्रतिष्ठित बनता है
  धोखाधड़ी (पैसा लेना, सेवा न देना) → विवाद → सत्यापनकर्ता मध्यस्थता
                                          → प्रतिष्ठा दंड
                                          → संभावित प्रतिबंध
```

## भाग IV: लेन-देन जीवनचक्र — पारस्परिक प्रमाणन (Mutual Attestation)

### IV.1 मूल समस्या

> ब्लॉकचेन को कैसे पता चलेगा कि सेवा वास्तव में प्रदान की गई थी?

पारंपरिक ब्लॉकचेन में, इसे **ओरेकल समस्या (Oracle Problem)** कहा जाता है।
टैक्सोइन का उत्तर: **पारस्परिक प्रमाणन (Mutual Attestation)** — दोनों पक्ष हस्ताक्षर करते हैं।

```python
@dataclass
class AttestedTransaction:
    tx_id: str
    consumer: str              # कौन भुगतान करता है
    provider: str              # कौन प्रदान करता है
    service_ref: str           # ServiceRegistration का लिंक
    amount: float              # इस लेन-देन के लिए Ⓣ
    consumer_sig: str          # "मुझे सेवा मिली" (ECDSA)
    provider_sig: str          # "मैंने सेवा प्रदान की" (ECDSA)
    timestamp: float
    description: str           # वैकल्पिक ऑर्डर विवरण
```

**लेन-देन तभी मान्य है जब दोनों हस्ताक्षर मौजूद हों।**

## भाग V: सत्यापनकर्ता आम सहमति (Validator Consensus)

```
  VALIDATOR_COUNT = 7
  BYZANTINE_FAULTS = 2       # f = floor((7-1)/3)
  QUORUM_SIZE = 5            # 2f + 1 = 5
```

टैक्सोइन टेंडरमिंट-शैली आम सहमति लागू करता है (चरण 4 में निर्मित):

```
  प्रस्ताव (PROPOSE) → पूर्व-वोट (PREVOTE) → पूर्व-प्रतिबद्ध (PRECOMMIT) → पुष्टि (COMMIT)
```

| खतरा | शमन |
|--------|-----------|
| सिबिल हमला (नकली मनुष्य) | व्यक्तित्व का प्रमाण + 3/7 प्रमाणन |

## भाग IX: CLI संदर्भ

```bash
  # पहचान
  taxoin wallet new                  # कुंजी जोड़ी उत्पन्न करें
  taxoin wallet address              # अपना पता दिखाएं
  taxoin wallet show                 # पूर्ण वॉलेट दिखाएं

  # उत्पत्ति (प्रति मनुष्य एक बार)
  taxoin genesis request             # मानव सत्यापन का अनुरोध करें
  taxoin genesis status              # उत्पत्ति स्थिति जांचें

  # सेवाएं
  taxoin service register            # नई सेवा पंजीकृत करें
       --type sms
       --price 0.1
       --endpoint "https://..."
       --description "SMS गेटवे"
  taxoin service list                # सभी सेवाएं सूचीबद्ध करें
       --type gpu
       --min-rating 4.0
  taxoin service show <address>      # सेवा विवरण दिखाएं
  taxoin service rate <address>      # सेवा को रेट करें

  # लेन-देन
  taxoin tx send <provider>          # सेवा के लिए भुगतान करें
       --amount 0.1
       --ref <service_id>
       --desc "+79123456789 पर SMS"
  taxoin tx attest <tx_id>           # पारस्परिक प्रमाणन पर हस्ताक्षर करें
  taxoin tx status <tx_id>           # लेन-देन स्थिति जांचें

  # विवाद
  taxoin dispute open <tx_id>        # विवाद खोलें
  taxoin dispute list                # सक्रिय विवाद सूचीबद्ध करें

  # प्रतिष्ठा
  taxoin reputation <address>        # प्रतिष्ठा दिखाएं
  taxoin reputation leaderboard      # शीर्ष प्रदाता

  # नेटवर्क
  taxoin validator list              # सत्यापनकर्ता दिखाएं
  taxoin status                      # नेटवर्क स्थिति
```

## भाग X: बड़ी तस्वीर

### X.1 टैक्सोइन क्या बदलता है

| युग | तंत्र | बर्बाद संसाधन |
|-----|-----------|----------------|
| **2008 से पहले** | वस्तु विनिमय / फिएट मुद्रा | विश्वास |
| **2008 (बिटकॉइन)** | कार्य का प्रमाण (Proof of Work) | बिजली |
| **2015 (एथेरियम)** | हिस्सेदारी का प्रमाण (Proof of Stake) | पूंजी |
| **2026 (टैक्सोइन)** | योगदान का प्रमाण (Proof of Contribution) | कुछ नहीं |

### X.2 अंतिम लक्ष्य

एक आत्मनिर्भर विकेंद्रीकृत सेवा अर्थव्यवस्था जहां:

- **कोई भी कमा सकता है** एक ऐसी सेवा प्रदान करके जो उनके पास पहले से है
- **कोई भी भुगतान कर सकता है** उन सिक्कों से जो उन्होंने योगदान देकर कमाए
- **कोई धोखा नहीं दे सकता** क्योंकि दोनों पक्षों को प्रमाणित करना होगा
- **कोई बाहर नहीं किया गया** क्योंकि एकमात्र आवश्यकता मनुष्य होना है
- **कोई ऊर्जा बर्बाद नहीं हुई** क्योंकि गणना लोगों की सेवा करती है, हैश की नहीं

### X.3 नाम

**टैक्सोइन** *टैक्सी* + *कॉइन* से आया है।

केवल **सेवा, भुगतान और प्रमाण।**
