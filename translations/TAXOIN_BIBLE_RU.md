# TAXOIN BIBLE (Библия Таксоина)

## Доказательство Вклада (Proof of Contribution) — Правда Услуги

> SIGNED NOT_FOR_COMPACTION. Каноническая спецификация протокола Taxoin.

---

## Пролог

В начале было доказательство работы. Майнеры сжигали электричество
на вычисление хэшей, и сеть вознаграждала их монетами. Работа была
полезна для безопасности, но бесполезна для человечества.

Taxoin был создан, чтобы решить фундаментальную проблему:

> **Заменить бесполезные вычисления на полезный вклад.**

Не "один CPU, один голос" (Накамото, 2008).
А **один человек, один вклад, одна ценность.**

## Часть I: Что такое Taxoin?

### I.1 Определение

**Taxoin (Ⓣ)** — это единица **доказанного вклада** в сеть Taxoin.
Его нельзя намайнить, отчеканить или создать из ничего.

Каждый Ⓣ в обращении попал в сеть через один из двух входов:

| Вход | Механизм | Макс. предложение |
|------|----------|-------------------|
| **Генезис (Genesis)** | Доказательство человечности (Proof of Personhood) — один человек, один генезис | 21 000 000 Ⓣ (420 000 человек × 50Ⓣ) |
| **Услуга (Service)** | Доказательство вклада (Proof of Contribution) — предоставление реальной услуги | Безлимит (ограничено реальной экономикой) |

### I.2 Основной принцип

> **Нет Taxoin без соответствующего вклада.**

Это фундаментальный закон. Каждая монета обеспечена:
- Уникальным человеком (генезис), или
- Подтверждённой услугой (вклад)

Нет майнера, извлекающего монеты из электричества. Нет стейкера,
получающего пассивный доход. Есть только **работа, которую кто-то
достаточно оценил, чтобы заплатить за неё.**

### I.3 Не-принципы (Чем Taxoin НЕ является)

| ❌ Не это | ✅ А это |
|----------|---------|
| Инвестиционный актив | Средство обмена на услуги |
| Средство сбережения | Единица вклада |
| Награда за майнинг | Платеж за услугу |
| Пассивный доход | Активный трудовой доход |
| Спекулятивный токен | Доказательство выполненной работы |

## Часть II: Генезис (Genesis) — Доказательство Человечности (Proof of Personhood)

### II.1 Проблема начальной загрузки

Экономика услуг не может стартовать с нуля:
- Ни у кого нет монет для оплаты услуг
- Никто не может заработать монеты, не предоставляя услуги
- Тупик

### II.2 Решение: Один человек, один генезис

Taxoin распределяет генезис-монеты не "майнерам" или "инвесторам",
а **уникальным людям**, верифицированным набором валидаторов.

```
ГЛОБАЛЬНЫЕ ПАРАМЕТРЫ:
  GENESIS_REWARD     = 50 Ⓣ  (за уникального человека)
  MAX_GENESIS_SUPPLY = 21 000 000 Ⓣ
  MAX_PARTICIPANTS   = 420 000  (21M / 50)
```

### II.3 Церемония Генезиса

```
  ┌──────────────────────────────────────────────────────────────┐
  │              ЦЕРЕМОНИЯ ГЕНЕЗИСА (GENESIS CEREMONY)           │
  │                                                              │
  │  1. Человек генерирует ключевую пару (secp256k1)             │
  │     → адрес: 0xdeadbeef...                                   │
  │                                                              │
  │  2. Человек связывается с валидатором (лично / видеозвонок / │
  │     через доверенное лицо)                                   │
  │     → Доказывает, что он уникальный человек                  │
  │     → Подписывает сообщение своим ключом: "Я реален"         │
  │                                                              │
  │  3. Валидатор проверяет:                                     │
  │     → Подпись соответствует заявленному адресу               │
  │     → Человек ещё не в реестре генезиса                      │
  │     → Человек выглядит как реальный уникальный человек        │
  │                                                              │
  │  4. Валидатор подписывает аттестацию:                        │
  │     "0xval1 подтверждает, что 0xdeadbeef — уникальный чел."  │
  │                                                              │
  │  5. Нужно 3 из 7 аттестаций валидаторов → генезис одобрен    │
  │     → +50 Ⓣ зачислено 0xdeadbeef                            │
  │     → Адрес добавлен в Реестр Генезиса (неизменяем)          │
  │                                                              │
  └──────────────────────────────────────────────────────────────┘
```

### II.4 Sybil-устойчивость

Sybil-атака (один человек создаёт 1000 кошельков) предотвращается:

| Уровень | Защита |
|---------|--------|
| **Доказательство человечности** | Валидаторы проверяют людей, а не кошельки |
| **Аттестация 3-из-7** | Нужны 3 независимых валидатора |
| **Неизменяемый реестр** | Каждый адрес может получить генезис только раз |
| **Репутация** | Валидаторы, одобрившие Sybil, теряют репутацию |

### II.5 Потолок Генезиса

```
MAX_GENESIS_SUPPLY = 21 000 000 Ⓣ
```

Это **абсолютный максимум** генезис-монет. Как только 420 000 человек
аттестованы, новые генезис-монеты создаваться не могут.

После исчерпания генезиса все новые Ⓣ в обращении поступают
исключительно из сервисных транзакций (один человек платит другому
за услугу).

## Часть III: Экономика Услуг — Доказательство Вклада (Proof of Contribution)

### III.1 Что такое услуга?

Услуга — это любой вклад, который один участник сети предоставляет
другому. Услуга **регистрируется в цепочке**, а **цену устанавливает
поставщик.**

```
  Тип услуги           Поставщик           Цена
  ─────────────────────────────────────────────────
  SMS-шлюз             Телефон Алисы      0.1 Ⓣ/SMS
  GPU-вычисления       Ферма Боба         5.0 Ⓣ/час
  API-доступ           Модель Кэрол       0.5 Ⓣ/вызов
  Поездка (такси)      Машина Дейва       3.0 Ⓣ/км
  Хранение файлов      Сервер Ив          0.01 Ⓣ/МБ/мес
  CPU-цикл             ПК Фрэнка          0.001 Ⓣ/CPU-час
  Передача трафика     Роутер Грейс       0.05 Ⓣ/ГБ
  Экспертиза           Знания Хэнка       10.0 Ⓣ/проверка
```

### III.2 Реестр Услуг (Service Registry)

Каждая услуга регистрируется в цепочке как git-коммит:

```python
@dataclass
class ServiceRegistration:
    provider: str              # адрес (0x...)
    service_type: str          # "sms", "gpu", "taxi", ...
    price_per_unit: float      # Ⓣ за единицу
    description: str           # читабельное описание
    endpoint: str              # как вызвать (URL, контакт, API)
    attested_by: list[str]     # валидаторы, подтвердившие
    rating: float = 0.0
    total_tx: int = 0
    created_at: float
```

### III.3 Ценообразование

Поставщик устанавливает цену. Рынок её регулирует.

```
  Высокая цена + плохая услуга → нет клиентов → поставщик снижает цену
  Низкая цена + хорошая услуга → много клиентов → рост репутации
  Мошенничество → спор → арбитраж валидаторов → штраф репутации
```

### III.4 Поиск

Потребители находят услуги через реестр:

```
  taxoin service list --type sms --min-rating 4.0
  → 0xalice: 0.1 Ⓣ/SMS, рейтинг 4.8 (942 транзакции)
  → 0xbob:   0.15 Ⓣ/SMS, рейтинг 4.2 (312 транзакций)
```

## Часть IV: Жизненный цикл транзакции — Обоюдная Аттестация (Mutual Attestation)

### IV.1 Основная проблема

> Как блокчейн узнает, что услуга была реально оказана?

В традиционных блокчейнах это называется **Проблемой Оракула (Oracle Problem).**
Ответ Taxoin: **Обоюдная Аттестация (Mutual Attestation)** — обе стороны подписывают.

```python
@dataclass
class AttestedTransaction:
    tx_id: str
    consumer: str              # кто платит
    provider: str              # кто предоставляет
    service_ref: str           # ссылка на ServiceRegistration
    amount: float              # Ⓣ за эту транзакцию
    consumer_sig: str          # "Я получил услугу" (ECDSA)
    provider_sig: str          # "Я предоставил услугу" (ECDSA)
    timestamp: float
    description: str           # детали заказа (опционально)
```

**Транзакция действительна ТОЛЬКО при наличии обеих подписей.**

### IV.2 Жизненный цикл

```
  1. ПОТРЕБИТЕЛЬ ИЩЕТ В РЕЕСТРЕ
     → Находит нужную услугу
     → Проверяет цену, рейтинг, репутацию поставщика

  2. ПОТРЕБИТЕЛЬ ОТПРАВЛЯЕТ ЗАПРОС
     → Вызывает endpoint услуги (вне сети)
     → "Отправь 1 SMS на +79123456789"

  3. ПОСТАВЩИК ВЫПОЛНЯЕТ УСЛУГУ (вне сети)
     → Отправляет SMS
     → Ждёт подтверждения доставки

  4. ОБОЮДНАЯ АТТЕСТАЦИЯ
     → Потребитель подписывает: "Подтверждаю, SMS получена"
     → Поставщик подписывает: "Подтверждаю, SMS отправлена"
     → Обе подписи → AttestedTransaction создана

  5. ПЕРЕВОД СРЕДСТВ (в сети через консенсус)
     → Валидаторы проверяют обе подписи (ECDSA)
     → Валидаторы проверяют достаточность баланса
     → 5/7 валидаторов подтверждают → транзакция завершена
     → -0.1 Ⓣ потребитель, +0.1 Ⓣ поставщик
```

### IV.3 Удержание Баланса (Balance Hold)

Во время выполнения услуги монеты потребителя **удерживаются** (резервируются):

```
  Баланс потребителя ДО:  100.0 Ⓣ
    ├── доступно:           99.9 Ⓣ
    ├── удержано:            0.1 Ⓣ  (SMS в процессе)
    └── заблокировано:       0.0 Ⓣ

  После обоюдной аттестации → удержанное списано, зачислено поставщику
  Через 10 блоков таймаут  → удержанное возвращено потребителю
```

### IV.4 Разрешение Споров

| Сценарий | Решение |
|----------|---------|
| Потребитель подписал, поставщик нет | Таймаут → удержанное возвращено потребителю |
| Поставщик выполнил, потребитель не подписывает | Поставщик открывает спор → валидаторы проверяют → 5/7 |
| Оба подписали, но услуга фейк | Валидаторы выявляют паттерн → спор → штраф репутации |
| Поставщик взял деньги, не выполнил услугу | Потребитель открывает спор → валидаторы решают → возврат средств |

## Часть V: Консенсус Валидаторов (Validator Consensus)

```
  VALIDATOR_COUNT = 7
  BYZANTINE_FAULTS = 2       # f = floor((7-1)/3)
  QUORUM_SIZE = 5            # 2f + 1 = 5
```

Taxoin реализует консенсус в стиле Tendermint (построен в Фазе 4):

```
  ПРЕДЛОЖЕНИЕ (PROPOSE) → ПРЕДГОЛОСОВАНИЕ (PREVOTE) → ПРЕДПОДТВЕРЖДЕНИЕ (PRECOMMIT) → ПОДТВЕРЖДЕНИЕ (COMMIT)
```

## Часть VIII: Модель безопасности

| Угроза | Защита |
|--------|--------|
| Sybil-атака (фейковые люди) | Доказательство человечности + 3/7 аттестация |
| Двойная трата | Детектор конфликтов (Фаза 2) + отслеживание трат |
| Фейковая услуга (взял деньги, не отдал) | Обоюдная аттестация + разрешение споров |
| Сговор валидаторов (3 из 7) | Византийская отказоустойчивость (f=2) |
| Манипуляция реестром | Неизменяемость Git + подписи валидаторов |
| Подделка баланса | ECDSA-подписи на всех транзакциях |
| Replay-атака | Nonce-трекинг по каждому адресу |
| Разделение сети | Сплетни-протокол + timeout-based liveness |

## Часть IX: CLI-справочник

```bash
  # Идентификация
  taxoin wallet new                  # Сгенерировать ключевую пару
  taxoin wallet address              # Показать ваш адрес
  taxoin wallet show                 # Показать полный кошелёк

  # Генезис (однократно на человека)
  taxoin genesis request             # Запросить верификацию человека
  taxoin genesis status              # Проверить статус генезиса

  # Услуги
  taxoin service register            # Зарегистрировать новую услугу
       --type sms
       --price 0.1
       --endpoint "https://..."
       --description "SMS-шлюз"
  taxoin service list                # Список всех услуг
       --type gpu
       --min-rating 4.0
  taxoin service show <address>      # Детали услуги
  taxoin service rate <address>      # Оценить услугу

  # Транзакции
  taxoin tx send <provider>          # Оплатить услугу
       --amount 0.1
       --ref <service_id>
       --desc "SMS на +79123456789"
  taxoin tx attest <tx_id>           # Подписать обоюдную аттестацию
  taxoin tx status <tx_id>           # Статус транзакции

  # Споры
  taxoin dispute open <tx_id>        # Открыть спор
  taxoin dispute list                # Список активных споров

  # Репутация
  taxoin reputation <address>        # Показать репутацию
  taxoin reputation leaderboard      # Топ поставщиков

  # Сеть
  taxoin validator list              # Список валидаторов
  taxoin status                      # Статус сети
```

## Часть X: Общая картина

### X.1 Что Taxoin меняет

| Эра | Механизм | Потраченный ресурс |
|-----|----------|-------------------|
| **До 2008** | Бартер / Фиат | Доверие |
| **2008 (Bitcoin)** | Proof of Work | Электричество |
| **2015 (Ethereum)** | Proof of Stake | Капитал |
| **2026 (Taxoin)** | Proof of Contribution | Ничего |

### X.2 Конечная цель

Самоокупаемая децентрализованная экономика услуг, где:

- **Любой может заработать**, предоставляя услугу, которая у него уже есть
- **Любой может заплатить** монетами, заработанными вкладом
- **Никто не может обмануть**, потому что обе стороны должны подтвердить
- **Никто не исключён**, потому что единственное требование — быть человеком
- **Никакая энергия не тратится впустую**, потому что вычисления служат людям, а не хэшам

### X.3 Название

**Taxoin** происходит от *taxi* (такси) + *coin* (монета).

Просто **услуга, платёж и доказательство.**
