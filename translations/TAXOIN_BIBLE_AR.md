## مقدمة

في البداية، كان إثبات العمل. كان المعدنون يحرقون الكهرباء لحساب التجزئات (Hashes)،
وكانت الشبكة تكافئهم بالعملات. كان العمل مفيدًا للأمان، لكنه عديم الفائدة للبشرية.

تم إنشاء تاكسوين لحل مشكلة أساسية واحدة:

> **استبدال الحساب عديم الفائدة بالمساهمة المفيدة.**

ليس "وحدة معالجة مركزية واحدة، صوت واحد" (ناكاموتو، 2008).
ولكن **إنسان واحد، مساهمة واحدة، قيمة واحدة.**

---

## الجزء الأول: ما هو التاكسوين؟

### I.1 تعريف

**التاكسوين (Ⓣ)** هو وحدة **المساهمة المثبتة** لشبكة تاكسوين.
لا يمكن تعدينه أو سكه أو إنشاؤه من لا شيء.

كل Ⓣ متداول دخل الشبكة عبر إحدى بوابتين:

| البوابة | الآلية | الحد الأقصى للإمداد |
|------|-----------|------------|
| **النشأة (Genesis)** | إثبات الشخصية (Proof of Personhood) — إنسان واحد، نشأة واحدة | 21,000,000 Ⓣ (420,000 شخص × 50Ⓣ) |
| **الخدمة (Service)** | إثبات المساهمة (Proof of Contribution) — تقديم خدمة حقيقية | غير محدود (محدود بالنشاط الاقتصادي الحقيقي) |

### I.2 المبدأ الأساسي

> **لا يوجد تاكسوين دون مساهمة مقابلة.**

هذا هو القانون الأساسي. كل عملة مدعومة بـ:
- إنسان فريد (النشأة)، أو
- خدمة محققة مقدمة (مساهمة)

لا يوجد معدن يستخرج العملات من الكهرباء. لا يوجد مراهن يكسب دخلاً سلبياً.
يوجد فقط **عمل قام شخص ما بتقييمه بما يكفي للدفع مقابله.**

## الجزء الثاني: النشأة (Genesis) — إثبات الشخصية (Proof of Personhood)

### II.1 مشكلة البدء

لا يمكن لاقتصاد الخدمات أن يبدأ من الصفر:
- لا أحد لديه عملات لدفع ثمن الخدمات
- لا أحد يمكنه كسب العملات دون تقديم خدمات
- طريق مسدود

### II.2 الحل: إنسان واحد، نشأة واحدة

يوزع تاكسوين عملات النشأة ليس لـ"المعدنين" أو "المستثمرين"
بل لـ**البشر الفريدين** الذين تم التحقق منهم من قبل مجموعة المدققين.

```
المعلمات العالمية:
  GENESIS_REWARD     = 50 Ⓣ  (لكل شخص فريد)
  MAX_GENESIS_SUPPLY = 21,000,000 Ⓣ
  MAX_PARTICIPANTS   = 420,000  (21M / 50)
```

### II.3 حفل النشأة

```
  ┌──────────────────────────────────────────────────────────────┐
  │               حفل النشأة (GENESIS CEREMONY)                   │
  │                                                              │
  │  1. يولد الإنسان زوج مفاتيح (secp256k1)                      │
  │     → العنوان: 0xdeadbeef...                                 │
  │                                                              │
  │  2. يتصل الإنسان بمدقق (شخصيًا / مكالمة فيديو / من خلال      │
  │     مرجع موثوق)                                              │
  │     → يثبت أنه إنسان فريد                                    │
  │     → يوقع رسالة بمفتاحه: "أنا حقيقي"                       │
  │                                                              │
  │  3. يتحقق المدقق:                                           │
  │     → التوقيع يطابق العنوان المعلن                           │
  │     → الإنسان ليس مسجلاً بالفعل في سجل النشأة                │
  │     → يبدو الإنسان شخصًا حقيقيًا وفريدًا                    │
  │                                                              │
  │  4. يوقع المدقق على مصادقة:                                 │
  │     "0xval1 يصادق أن 0xdeadbeef إنسان فريد"                 │
  │                                                              │
  │  5. يحتاج 3 من 7 مصادقات المدققين → الموافقة على النشأة      │
  │     → +50 Ⓣ مضاف إلى 0xdeadbeef                             │
  │     → إضافة العنوان إلى سجل النشأة (غير قابل للتغيير)        │
  │                                                              │
  └──────────────────────────────────────────────────────────────┘
```

## الجزء الثالث: اقتصاد الخدمات — إثبات المساهمة (Proof of Contribution)

### III.1 ما هي الخدمة؟

الخدمة هي أي مساهمة يقدمها أحد المشاركين في الشبكة لآخر.
يتم **تسجيل الخدمة على السلسلة** و**تسعيرها من قبل المزود.**

```python
@dataclass
class ServiceRegistration:
    provider: str              # العنوان (0x...)
    service_type: str          # "sms", "gpu", "taxi", ...
    price_per_unit: float      # Ⓣ لكل وحدة
    description: str           # مقروء للبشر
    endpoint: str              # كيفية الاتصال (URL, جهة اتصال, API)
    attested_by: list[str]     # المدققون الذين تحققوا
    rating: float = 0.0
    total_tx: int = 0
    created_at: float
```

## الجزء الرابع: دورة حياة المعاملة — المصادقة المتبادلة (Mutual Attestation)

### IV.1 المشكلة الأساسية

> كيف تعرف blockchain أن الخدمة تم تقديمها بالفعل؟

في سلاسل الكتل التقليدية، يسمى هذا **مشكلة التوقع (Oracle Problem).**
إجابة تاكسوين: **المصادقة المتبادلة (Mutual Attestation)** — يوقع كلا الطرفين.

```python
@dataclass
class AttestedTransaction:
    tx_id: str
    consumer: str              # من يدفع
    provider: str              # من يقدم
    service_ref: str           # رابط لـ ServiceRegistration
    amount: float              # Ⓣ لهذه المعاملة
    consumer_sig: str          # "استلمت الخدمة" (ECDSA)
    provider_sig: str          # "قدمت الخدمة" (ECDSA)
    timestamp: float
    description: str           # تفاصيل الطلب (اختياري)
```

**المعاملة صالحة فقط إذا كان كلا التوقيعين موجودين.**

```
  VALIDATOR_COUNT = 7
```

## الجزء السابع: اقتصاد الرموز (Tokenomics)

### VII.1 إجمالي الإمداد

```
  MAX_GENESIS_SUPPLY   = 21,000,000 Ⓣ  (حد أقصى صارم)
  MAX_SERVICE_SUPPLY   = غير محدود      (محدود بالاقتصاد الحقيقي)
```

### VII.2 توزيع النشأة

```
  420,000 إنسان × 50 Ⓣ = 21,000,000 Ⓣ
  ──────────────────────────────────────
  لا تعدين مسبق. لا طرح أولي للعملة. لا تخصيص داخلي.
  كل إنسان يحصل على نفس المبلغ تمامًا.
```

## الجزء الثامن: نموذج الأمان

| التهديد | التخفيف |
|--------|-----------|
| هجوم Sybil (بشر مزيفون) | إثبات الشخصية + مصادقة 3/7 |
| الإنفاق المزدوج | كاشف التعارض (المرحلة 2) + تتبع الإنفاق |
| خدمة مزيفة (أخذ المال، عدم التسليم) | المصادقة المتبادلة + حل النزاعات |
| تواطؤ المدققين (3 من 7 ضارون) | تحمل الأخطاء البيزنطية (f=2) |
| التلاعب بالسجل | ثبات Git + توقيعات المدققين |
| تزوير الرصيد | توقيعات ECDSA على جميع المعاملات |
| هجوم إعادة التشغيل | تتبع Nonce لكل عنوان |
| تقسيم الشبكة | بروتوكول النميمة + الحيوية المستندة إلى المهلة |

## الجزء التاسع: مرجع واجهة الأوامر (CLI)

```bash
  # الهوية
  taxoin wallet new                  # إنشاء زوج مفاتيح
  taxoin wallet address              # عرض عنوانك
  taxoin wallet show                 # عرض المحفظة كاملة

  # النشأة (مرة واحدة لكل إنسان)
  taxoin genesis request             # طلب التحقق البشري
  taxoin genesis status              # التحقق من حالة النشأة

  # الخدمات
  taxoin service register            # تسجيل خدمة جديدة
       --type sms
       --price 0.1
       --endpoint "https://..."
       --description "بوابة SMS"
  taxoin service list                # عرض جميع الخدمات
       --type gpu
       --min-rating 4.0
  taxoin service show <address>      # عرض تفاصيل الخدمة
  taxoin service rate <address>      # تقييم الخدمة

  # المعاملات
  taxoin tx send <provider>          # الدفع مقابل خدمة
       --amount 0.1
       --ref <service_id>
       --desc "SMS إلى +79123456789"
  taxoin tx attest <tx_id>           # التوقيع على المصادقة المتبادلة
  taxoin tx status <tx_id>           # التحقق من حالة المعاملة

  # النزاعات
  taxoin dispute open <tx_id>        # فتح نزاع
  taxoin dispute list                # عرض النزاعات النشطة

  # السمعة
  taxoin reputation <address>        # عرض السمعة
  taxoin reputation leaderboard      # أفضل المزودين

  # الشبكة
  taxoin validator list              # عرض المدققين
  taxoin status                      # حالة الشبكة
```

## الجزء العاشر: الصورة الكبيرة

### X.1 ما يغيره تاكسوين

| العصر | الآلية | المورد المهدر |
|-----|-----------|----------------|
| **قبل 2008** | المقايضة / العملة الورقية | الثقة |
| **2008 (بيتكوين)** | إثبات العمل (Proof of Work) | الكهرباء |
| **2015 (إيثريوم)** | إثبات الحصة (Proof of Stake) | رأس المال |
| **2026 (تاكسوين)** | إثبات المساهمة (Proof of Contribution) | لا شيء |

### X.2 الهدف النهائي

اقتصاد خدمات لامركزي ومستدام ذاتيًا حيث:

- **أي شخص يمكنه الكسب** من خلال تقديم خدمة يمتلكها بالفعل
- **أي شخص يمكنه الدفع** بعملات كسبها من خلال المساهمة
- **لا يمكن لأحد الغش** لأنه يجب على كلا الطرفين المصادقة
- **لا يتم استبعاد أحد** لأن الشرط الوحيد هو أن تكون إنسانًا
- **لا يتم إهدار طاقة** لأن الحوسبة تخدم الناس، وليس التجزئات

### X.3 الاسم

**تاكسوين** مشتق من *تاكسي* + *كوين*.

مجرد **خدمة ودفع وإثبات.**
