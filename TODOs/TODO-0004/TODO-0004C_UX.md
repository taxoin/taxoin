# TODO-0004C: UX — Seamless Mutual Attestation

**Цель:** Сделать процесс mutual attestation удобным для пользователя.

**Ссылка:** RESEARCH.md → Раздел 2

**Проблема:** Обмен подписями (consumer_sig + provider_sig) между двумя
сторонами должен происходить без трения. Текущий CLI требует ручного
копирования tx_id и подписей.

**Возможные решения:**
1. **Direct messaging** — provider отправляет unsigned tx consumer'у,
   consumer подписывает, возвращает, provider добавляет свою подпись
   и отправляет в сеть
2. **Shared endpoint** — обе стороны вызывают один API endpoint
   для одновременной подписи
3. **Off-chain relay** — простой relay-сервер для обмена подписями
   (опционально, не обязательно децентрализованный для этого шага)

**Задачи:**
- [ ] CLI: `tx send` выводит QR-код или ссылку для consumer
- [ ] CLI: consumer подписывает одним нажатием
- [ ] CLI: provider получает уведомление о подписи consumer
- [ ] Опционально: HTTP relay для обмена подписями
